Overview

In recent years there has been a rapid expansion of Javascript’s ecosystem. To this fact has contributed both the establishment of the internet as a development platform for web applications as well as the Node.js platform, which offered the opportunity to use the language outside the browser, for the development of any kind of application. Node’s success lead to the creation of the first package manager for JavaScript, which now hosts tens of thousands of JavaScript packages. The accumulation of a large number of software packages in a relatively short time has made the selection and detection of valuable packages difficult. The large volume of information and data available both in repositories and source code could be utilized for the improvement and evaluation of the available software. The application aims to enhance the software package selection process through the use of quality metrics for their assessment. Although we currently only use static analysis metrics we want to expand the information associated with each package using data from their repositories and issue trackers or a combination of them.

Features

The application is divided into the following sections:

  • Analytics. You can search for npm packages and see their profiles filled with software quality metrics extracted by our analyzers.
  • Comparison. You can compare two packages side by side based on some major metrics.
  • Visualization. You can visualize and interact with the dependency graphs.
The app is in alpha state and things might not work as expected.

Internals

The following diagram demonstrates the current architecture of the application.

The four major components are:

  • Web Server. A Node.js (hapi) server with an Angular frontend.
  • Static analysis services:
    • eslint
    • escomplex
    • jsinspect
  • Graph database. It is used to better examine the dependency graph, leveraging the Tinkerpop stack.
  • NoSQL database. A couchDB instance that holds a copy of the npm registry and the results of the static analyzers.

Every change on the official npm registry is propagated into our system so it remains up-to-date. The whole system is hosted in VMs provided by ~okeanos.
